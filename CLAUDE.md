## Project

**Name:** project-init
**Stack:** Mixed / to document
**Purpose:** `project-init` rationalizes project creation, enforces standards from day one, and automates the repetitive setup work that every new repository needs.

## Working Rules

- Language: English — all code, comments, documentation, instructions, and configuration files must be in English.
- Commits: Conventional Commits (`type(scope): description`).
- Prefer repository make targets when a Makefile is available.
- Read `.github/instructions/*.instructions.md` when present before starting task-specific work.

## Claude Compatibility

- Claude Code hooks are configured in `.claude/settings.json`.

- Keep repository-specific overrides in this file and keep generic automation in `.claude/`.

## Execution Standard

All projects generated by `project-init` must comply with `chrysa/shared-standards/EXECUTION_STANDARD.md`.

Key requirements enforced at generation time:
- **§1 Makefile** — all 13 required targets (help, install, dev, test, test-cov, lint, format, typecheck, build, docker-up, docker-down, clean, pre-commit)
- **§2 Directory** — standard layout with `.github/workflows/`, `docs/`, `CLAUDE.md`, `cliff.toml`, `GitVersion.yml`, `opencode.json`
- **§3 Branching** — Conventional Commits enforced via `.pre-commit-config.yaml`
- **§4 Testing** — 80% line coverage, deterministic, `test_<unit>_when_<cond>_should_<expected>` naming, `coverage.xml` on every CI run
- **§5 CI/CD** — `ci-*.yml` → labeler → PR review → `release.yml` → `pages.yml`

Deviations from the standard require a documented ADR.
